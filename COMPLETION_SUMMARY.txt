================================================================================
KEY INNOVATION: STATISTICAL SAMPLE SELECTION
================================================================================

The main ML improvement requested:

PROBLEM:
  Training only on sets A+B+C doesn't generalize to Set D because:
  - Different network infrastructure
  - Time-dependent patterns
  - Domain shift

SOLUTION: Distance-from-Centroid Sampling
  1. Compute centroid of A+B+C normal traffic in feature space
  2. Calculate distance of each Set D normal sample to centroid
  3. Select N farthest samples (most diverse/different)
  4. Use these diverse samples to augment training pool

RESULT:
  - Better generalization to unseen network conditions
  - Improved model robustness
  - Addresses domain shift problem
  - 2 implementations: distance-from-centroid (default) + KMeans

ALTERNATIVE: KMeans Clustering
  - Cluster Set D normals into K groups
  - Select representative samples from each cluster
  - Balances coverage across different normal patterns

================================================================================
MODEL CONFIGURATION
================================================================================

Algorithm: RandomForestClassifier
  - n_estimators: 100
  - max_depth: 15
  - min_samples_leaf: 2
  - class_weight: "balanced" (handles class imbalance)
  - random_state: 42 (reproducible)
  - n_jobs: -1 (parallel)

Class Weighting:
  w_c = N / (2 * n_c)
  - Ensures equal training focus on minority class (DDoS attacks)
  - Prevents model from ignoring rare attacks

Preprocessing:
  - Numeric: Median imputation → StandardScaler
  - Categorical: Most frequent imputation → OneHotEncoder
  - Handles infinities and missing values
